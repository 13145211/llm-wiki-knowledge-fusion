# Bulk Backfill Guide

## When You Need This

You have:
- Hundreds or thousands of source summaries in `wiki/sources/`
- Empty `wiki/concepts/`, `wiki/entities/`, `wiki/methods/`
- No systematic extraction of entities, concepts, or methods

## Goal

Automatically extract entities, concepts, and methods from all source summaries and create the corresponding wiki pages.

## Step-by-Step

### Step 1: Prepare Environment

Ensure you have:
- Python 3.8+ with `pyyaml` and `markdown` packages
- LLM-Wiki directory structure complete (including `_templates/`)
- Backup of your existing wiki (git commit or copy)

### Step 2: Run Entity Extraction Script

```bash
python scripts/backfill_entities.py \
  --wiki-dir "F:\xz-wiki-upgraded\wiki" \
  --batch-size 100 \
  --dry-run  # First run with dry-run to preview
```

The script:
1. Scans all `wiki/sources/*.md` files
2. Parses YAML frontmatter for `authors`, `tags`, `methods`, `catalysts`, `materials` fields
3. For each unique value, creates a wiki page in the appropriate directory
4. Links all relevant sources to each entity/concept/method page
5. Updates `wiki/index.md`

### Step 3: Review and Clean

After dry-run, review the generated pages:
- Check for duplicates (e.g., `Cu-CeO2` vs `Cu/CeO2` vs `Cu-CeO2-catalyst`)
- Merge near-duplicates
- Verify author names are consistent (e.g., `John Smith` vs `J. Smith` vs `Smith, J.`)
- Check concept definitions make sense

### Step 4: Commit and Sync to GBrain

```bash
# Commit to git
cd F:\xz-wiki-upgraded
git add wiki/concepts wiki/entities wiki/methods wiki/index.md
git commit -m "backfill: extracted <count> entities from <count> sources"

# Sync to GBrain
python scripts/sync_to_gbrain.py --wiki-dir "F:\xz-wiki-upgraded\wiki"
```

## Backfill Script Specification

### Input

- `wiki/sources/*.md` — source summary files with YAML frontmatter

### Output

- `wiki/concepts/*.md` — concept pages
- `wiki/entities/*.md` — entity pages
- `wiki/methods/*.md` — method pages
- `wiki/index.md` — updated index

### Frontmatter Fields Used

| Field | Target Directory | Page Type |
|-------|-----------------|-----------|
| `authors` | `wiki/entities/` | `author` |
| `institutions` | `wiki/entities/` | `institution` |
| `catalysts` | `wiki/concepts/` | `catalyst` |
| `materials` | `wiki/concepts/` | `material` |
| `methods` | `wiki/methods/` | `method` |
| `tags` | `wiki/concepts/` | `concept` |

### Slug Generation Rules

1. Lowercase
2. Replace spaces with hyphens
3. Remove special characters: `()[]{}\/:*?"<>|`
4. For non-ASCII: use pinyin or English translation
5. Deduplicate: if slug exists, append `-2`, `-3`, etc.

### Page Content Generation

For each entity/concept/method, the script generates:

```markdown
---
title: <Name>
type: <page-type>
created: <YYYY-MM-DD>
updated: <YYYY-MM-DD>
sources: [<source-slug-1>, <source-slug-2>, ...]
tags: [<auto-extracted-tags>]
---

## Overview
<Auto-generated from source summaries: 2-3 sentences>

## Key Papers
- [[../sources/<slug-1>]] — <brief note from frontmatter>
- [[../sources/<slug-2>]] — <brief note from frontmatter>

## Related <Entities/Concepts/Methods>
<Auto-linked based on co-occurrence in sources>
```

### Deduplication

The script maintains a mapping file to handle duplicates:

```json
{
  "slugs": {
    "cu-ceo2": "Cu-CeO2",
    "cu-ceo2-catalyst": "Cu-CeO2",
    "cu-ce-o2": "Cu-CeO2"
  },
  "authors": {
    "john-smith": "John Smith",
    "j-smith": "John Smith",
    "smith-j": "John Smith"
  }
}
```

Review and edit this mapping after the first run.

## Manual Backfill (Small Libraries)

If you have < 100 sources, you can do manual backfill:

1. Read each source summary
2. Note unique authors, catalysts, methods, concepts
3. Create entity/concept/method pages manually
4. Link sources to each page

## Post-Backfill Checklist

- [ ] All concepts have definitions
- [ ] All entities have correct types
- [ ] All methods have descriptions
- [ ] No duplicate pages
- [ ] Index.md is up to date
- [ ] GBrain sync completed
- [ ] Git committed

## Common Issues

### Too many concepts

If `tags` extraction generates hundreds of concepts:
- Filter out generic tags (e.g., `catalysis`, `reaction`)
- Group related concepts (e.g., `Cu-CeO2`, `Cu/CeO2`, `copper-ceria` → `Cu-CeO2`)
- Use a domain-specific stoplist

### Author name variants

- Normalize before backfill: `Smith, J.` → `John Smith`
- Use ORCID or institution to disambiguate
- Manual review for common names

### Missing frontmatter fields

If sources lack `authors`, `methods`, etc.:
- Parse the markdown body for mentions
- Use NLP entity extraction (spaCy, etc.)
- Manual tagging for critical sources

## Performance

- 1000 sources → ~30 minutes (Python script)
- 10000 sources → ~4 hours (consider parallelization)
- Memory: ~2GB for 1000 sources
- Disk: ~10MB per 1000 generated pages

## Incremental Backfill

After initial backfill, run incrementally:

```bash
python scripts/backfill_entities.py \
  --wiki-dir "F:\xz-wiki-upgraded\wiki" \
  --incremental \
  --since "2026-07-01"
```

This only processes sources created/modified since the given date.
